Server-Side Folder Reorganization
What & Why
The server directory has 135 flat files (368 cross-imports) with no subdirectory grouping. Route files double as service layers — exporting core business logic consumed by other modules. This half of the reorganization focuses exclusively on the server side: extracting business logic, adding a path alias, moving files into domain subfolders, and slimming the 1,642-line god file (routes.ts) into a thin orchestrator.

Firm Rule Going Forward: All new server files must be placed in the appropriate domain subfolder — no more flat dumping into server/.

Done looks like
@server/ path alias added to tsconfig.json and vite.config.ts so all server imports use absolute paths
Business logic extracted out of route files into proper service files (aeo-grader, custom-auth, engine-registry, admin-revenue, crm)
server/routes.ts renamed to server/router.ts and slimmed to ~10 domain registration calls instead of 89 individual imports
Server files grouped into domain subfolders: routes/, services/, schedulers/, middleware/, migrations/, mcp/, core/
Barrel index.ts files in each subfolder re-exporting the public API (no cross-barrel imports)
replit_integrations/ upward relative imports converted to @server/ alias
All 368 relative imports converted to absolute @server/ paths
